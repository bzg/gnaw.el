: 3==3      <= A gnawed bone.

[[https://www.repostatus.org/#active][https://img.shields.io/badge/status-active-brightgreen.svg?style=for-the-badge]]
[[https://intver.org/][https://img.shields.io/badge/versioning-intver.org-blue.svg?style=for-the-badge]]

=gnaw.el= is an Emacs tool for [[https://codeberg.org/bzg/gnaw][gnaw]] and [[https://codeberg.org/bzg/bone][BONE]] with two faces:

- a *standalone browser* of BONE reports (=M-x gnaw=): list reports, read
  the email, view and apply patches, mark and filter -- no [[https://codeberg.org/bzg/gnaw][gnaw]] CLI
  required;
- the shared *data layer* of the mail front-ends [[https://codeberg.org/bzg/gnus-gnaw][gnus-gnaw]], [[https://codeberg.org/bzg/notmuch-gnaw][notmuch-gnaw]]
  and [[https://codeberg.org/bzg/mu4e-gnaw][mu4e-gnaw]].

It reads =config.edn= and =reports.json= directly, caches remote reports
and patches, and reads/writes =config.edn= and the =state.edn= shared with
the CLI, so state stays consistent across clients.

* Requirements

- Emacs 28.1 or later.
- =~/.config/gnaw/config.edn= with at least one source under =:sources=
  (=M-x gnaw= creates it interactively if missing).
- A published BONE instance in =0.9.2= format or later (the 0.9.3
  =meta.json= =reports-files= list is used when present).

* Installation

=gnaw.el= is not on MELPA yet. On Emacs 30 or later, install it straight
from its git repository with the built-in =package-vc=:

#+begin_src emacs-lisp
(use-package gnaw
  :vc (:url "https://codeberg.org/bzg/gnaw.el" :rev :newest))
#+end_src

Or interactively, without =use-package=:

: M-x package-vc-install RET https://codeberg.org/bzg/gnaw.el RET

The =transient= dependency is pulled from your regular package archives.
Update later with =M-x package-vc-upgrade RET gnaw RET=.

* Getting started

=M-x gnaw= opens the =*gnaw*= report browser, filling the frame. With no
source configured, it runs the full interactive setup (and repeats for
additional sources until you decline):

1. a source URL -- a base URL, a =reports/meta.json=, or a
   =reports/...json= file (all resolve to the =reports/= directory);
2. which report files to track, chosen from =meta.json='s =reports-files=
   (default =all-open.json=; several may be picked);
3. a source name (defaults to =meta.json='s =:source=);
4. a local git repository for that source, used to apply patches;
5. how to open report mail (=auto=, =mua=, =gnus=, =notmuch=, =mu4e= or =web=);
6. when =gnus= is chosen, the Gnus group where the mails are stored.

The source and patch repository are written to =config.edn=. The mail
client settings are saved through Customize. =M-x gnaw-configure= reruns
the full setup later on; =M-x gnaw-add-source= only adds or updates a
source.

* The =*gnaw*= browser

| Key   | Action                                                    |
|-------+-----------------------------------------------------------|
| =RET=   | open the report's email (see /Reading/ below)               |
| =SPC=   | toggle the report's email in a window below the list      |
| =f=     | follow mode: auto-show the email of the report at point   |
| =TAB=   | fold / unfold the series, or narrow to related reports    |
| =q=     | leave the related view, clear the filter, or quit         |
| =v=     | view an attachment (patches in =diff-mode=)                 |
| =s=     | save an attachment (proposes the configured =:repo=)        |
| =A=     | apply the report's patches with =git apply= (=C-u A=: =git am=) |
| =:=     | menu acting on the patches and attachments                |
| =/=     | filter the list by a query (see /Searching/)                |
| ===     | filter by one field (a =transient= menu)                    |
| =t=     | limit to a report type (bug, patch, request...)           |
| =T=     | filter by a topic, with completion                        |
| =a=     | only the acked reports (=acked:*=)                          |
| =o=     | only the owned reports (=owned:*=)                          |
| =c=     | only the closed reports (canceled, resolved...)           |
| =?=     | only the reports awaiting a reply                         |
| =~=     | only the reports with related reports                     |
| =+=     | only the reports with attachments                         |
| =C-RET= | toggle a filter on the cell at point (see /Searching/)      |
| =S=     | sort by the column at point; on Mark, by sticky/dismiss   |
| =^=     | sort by a criterion chosen from a menu (=gnaw-sort=)        |
| =!=     | mark sticky: bold + exported to todo.org, shown =!=         |
| =d=     | flag the report for dismissal, shown =D=                    |
| =x=     | dismiss all the flagged reports                           |
| =D=     | dismiss immediately: hide the report, shown =d=             |
| =u=     | remove the sticky/dismiss mark or the flag                |
| =_=     | show / hide dismissed reports                             |
| =h=     | list these bindings in a help window                      |
| =\=     | apply a preset filter (see =gnaw-preset-filters=)           |
| =g=     | reload from the local cache                               |
| =G=     | refresh the remote cache, then reload                     |

With a prefix argument, the filter keys -- =t=, =T=, =a=, =o=, =c=, =?=, =~=, =+=,
=C-RET=, and the fields of the === menu -- add their condition to the
active filter (AND) instead of replacing it: =a C-u o= keeps the
reports both acked and owned.

Patch series are folded into a single row (cover letter or first
patch) with a status summary like =(2 acked, 1 open)=; =TAB= unfolds the
series to list each patch, =TAB= again folds it back.

The Att column has three dedicated positions: =?= when the report
awaits a reply, =~= when it has related reports, then one attachment
glyph -- =+= for a single patch, =x= for several, =@= for calendar
events (=.ics=) or =#= for plain-text files, in that order of priority. The help echo of the
Flags and Att cells (=C-h .=, or hovering with the mouse) spells them
out.

On a report with related reports, =TAB= narrows the list to the report
and its relations, showing closed reports in italic like the BONE web
interface; =TAB= (or =q=) restores the full list. Related reports absent
from the loaded sources -- for instance closed reports when the
source is an open-reports JSON -- are shown as greyed-out placeholder
rows built from the relation metadata: type, subject, and the date
the relation was posed in the Date column; the help echo (=C-h .=, or
hovering) adds the relation kind, who posed it and the message-id.
=RET= on such a row still works when the mail client can look the
message up. Series
folding takes precedence: on a series member that also has related
reports, use =C-u TAB= to narrow. Setting or clearing a filter leaves
the related view.

Two mutually exclusive local marks drive the view: *sticky* promotes a
report (shown =!= in bold in the Mark column, and exported to =todo.org=
by the gnaw CLI), *dismiss* hides it. Dismissed reports are hidden by
default; =_= reveals them (shown =d= in italics). =d= only *flags* a report
(shown =D=) and =x= dismisses all the flagged ones at once, like in dired;
=D= dismisses right away. =u= clears the mark or flag. =D= and =x= turn off
the follow mode first. Marks live in =state.edn=, shared with the gnaw
CLI and the front-ends.

* Searching

=/= filters the list with a subset of the BONE web query syntax:
=key:value= tokens. Operators, loosest to tightest binding: =|= is OR
between clauses, space is AND between tokens, and a comma inside a
value is OR (=acked:alice,bob=). So =a b c | d= reads as =(a b c) | d=.

| Long                     | Short    | Matches                                   |
|--------------------------+----------+-------------------------------------------|
| =from:bzg=                 | =f:=       | from address or name                      |
| =subject:crash=            | =s:=       | subject (a bare word matches subject)     |
| =similar:a+b+c=            |          | subjects sharing 3 of the =+=-joined words  |
| =topic:agenda=             | =t:=       | topic                                     |
| =type:patch=               |          | report type                               |
| =priority:2=               | =p:=       | exact priority (0-3, shown A/B/C)         |
| =mid:<...>=                | =m:=       | message-id                                |
| =acked:bzg= =owned:= =closed:= | =a: o: c:= | by person (or =*= for any)                  |
| =urgent:= =important:=       | =u: i:=    | priority bits (set or not)                |
| =flags:AO=                 |          | Flags letters, all required (A O C R E S) |
| =att:~+=                   |          | Att glyphs, all required (? ~ + x @ #)    |
| =date:3d= =date:A..B=        | =d:=       | created (durations =Nd/Nw/Nm=, ranges)      |
| =deadline:2m=              | =D:=       | due within / on (forward)                 |
| =expired:10d=              | =e:=       | expiring within (forward)                 |

The active query shows in the mode line; =q= (or =C-u /=, or =/= then =RET=)
clears it.

=C-RET= builds the query from the cell at point: the author on From,
the report type on Type, created on or after that date on Created,
and =similar:= on Subject -- keeping the reports whose subject shares
at least three significant words (four letters or more, a hyphenated
name like =org-element= counting as one word; bracketed tags like
=[PATCH v2]= are ignored) with the subject at point.
=C-RET= again restores the view the filter replaced: the previous
query, a related-reports narrowing, or the full list.

* Reading messages

=gnaw-open-message-method= selects how =RET= opens a report, per source:
an alist mapping a source name to =auto=, =mua=, =gnus=, =notmuch=, =mu4e= or
=web= (the entry keyed by =t= is the default). =gnus= reads the group from
=gnaw-gnus-group= (an alist; asked with completion otherwise); =notmuch=
and =mu4e= open by message-id; =web= fetches the public-inbox archive. =SPC=
toggles the mail in a window below the list, keeping point in the
list; =f= (=gnaw-list-follow-mode=) does this automatically as point
moves.

Front-ends set =gnaw-open-message-function= to handle =auto=/=mua=. =M-x
gnaw-configure-email-client= configures this per source, including the
Gnus group when =gnus= is selected.

* Patches and attachments

=v= views an attachment and =s= saves one: with a single attachment
they act right away, otherwise they ask which one (with an /all
patches/ candidate on multi-patch reports). =A= applies the report's
patches with =git apply= (patch the working tree) and =C-u A= with
=git am= (apply as commits).

=:= opens the generic menu on the attachments of the report at point.
A single patch opens the patch menu: =v= views the patch in
=diff-mode=, =a= runs =git apply=, =m= runs =git am= and =s= saves the patch
file to a directory. A single calendar (=.ics=) or text attachment is
displayed right away. When the report has several attachments, =:=
lists them in a buffer where =+= (or =RET=) acts on the attachment at
point the same way; when several of them are patches, an extra first
row opens the patch menu on all of them at once.

Git runs in the source's =:repo= (falling back to =gnaw-apply-repo= or a
prompt), warning when a patch series looks incomplete. See =*gnaw-git*=
for the output; on a failed =git am=, run =git am --abort=. Saving
proposes the source's =:repo= as the target directory; a single patch
is saved there without prompting, and a prefix argument (=C-u s=) does
the same for several, overwriting silently.

* =config.edn=

#+begin_src clojure
{:sources [{:urls ["https://tracker.orgmode.org/reports/all-open.json"]
            :name "Org mode ML"
            :repo "~/src/org-mode"}]
 :my-addresses ["me@example.org"]
 :skip-columns ["owner"]}
#+end_src

- =:sources= -- each has =:urls= (a vector of reports.json URLs), an
  optional =:name= and a =:repo= for =git am=.
- =:my-addresses= -- your email addresses.
- =:skip-columns= -- browser columns to hide (by name, case-insensitive).

* Configuration

- =gnaw-config-dir= (default =~/.config/gnaw=): holds =config.edn=,
  =state.edn= and =cache/=. Sources always come from =config.edn=.
- =gnaw-open-message-method= / =gnaw-gnus-group=: per-source open method
  and Gnus group (see /Reading/).
- =gnaw-apply-repo=: fallback git repo for =git am=.
- =gnaw-after-update-hook=: run after =M-x gnaw-update=.
- =gnaw-list-sort-key= (default =("Created" . t)=): default sort column
  and direction; defaults to creation date, most recent first. Press
  =S= on a column to re-sort interactively.
- =gnaw-preset-filters= (default nil): list of predefined filter query
  strings; =\= prompts to apply one (=gnaw-select-preset-filter=).

* API for front-ends

| Function                        | Purpose                                    |
|---------------------------------+--------------------------------------------|
| =(gnaw-reports)=                  | collect open reports as =(MID . INFO)= pairs |
| =(gnaw-sources)=                  | resolved report URLs/paths from config.edn |
| =(gnaw-load-config)=              | plist =(:addresses :skip-columns :sources)=  |
| =(gnaw-read-state)=               | read =state.edn= into an alist               |
| =(gnaw-write-state STATE)=        | serialize an alist to =state.edn=            |
| =(gnaw-toggle-mark MID INFO ACT)= | toggle =:read=/=:todo=/=:sticky=, persist    |
| =(gnaw-action-on-p ST MID ACT)=   | is a mark currently set?                   |
| =(gnaw-normalize-mid MID)=        | ensure angle brackets around a message-id  |

INFO is a plist with =:type :subject :date :from :from-name= (used to
enrich state) plus display and action keys =:flags :priority :votes
:deadline :expiry :last-activity :topic :source :source-name
:archived-at :patches :events :texts :awaiting :related :series
:version :superseded-by=.
